# SaveTogether：用 Vue3 + DeepSeek 打造的家庭语音记账应用

最近做了一个家庭记账应用 SaveTogether，主要想解决两个人一起记账时的数据同步问题，顺便尝试了一下语音记账的功能。这篇文章记录一下开发过程中的技术选型和实现细节。

## 项目背景

家里一直有记账的习惯，但两个人分别记账时，数据同步是个问题。市面上很多记账 App 的多人协作功能需要付费会员，而且导出数据也要开 VIP，体验不太好。所以决定自己做一个，顺便试试用 AI 来简化记账流程。

前端用 Vue3 + Vite，状态管理用 Pinia，后端直接用 Supabase，省去了自己搭服务器的麻烦。数据可视化用了 ECharts，语音识别结合了浏览器原生的 Web Speech API 和 DeepSeek。

## 核心功能实现

### 1. 数据实时同步

用 Supabase 的 PostgreSQL Changes 功能实现多端实时同步：

```javascript
const channel = supabase
  .channel('table_changes')
  .on('postgres_changes', { event: '*', schema: 'public', table: 'table_name' }, (payload) => {
    // 处理数据变更
  })
  .subscribe()
```

这样一个人记账，另一边的设备能实时看到更新。

### 2. 语音记账

语音记账是这个项目的一个尝试性功能，流程是：语音识别 → 文本解析 → 保存账单。

#### 语音识别

用浏览器原生的 Web Speech API：

```javascript
const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition
const recognition = new SpeechRecognition()
recognition.lang = 'zh-CN'
recognition.continuous = false
recognition.onresult = (event) => {
  const transcript = event.results[0][0].transcript
  // 解析识别结果
}
```

#### 文本解析

先做本地规则匹配，再调用 DeepSeek API 做智能解析。本地规则定义了 19 个分类，每个分类有对应的关键词：

```javascript
const DEFAULT_VOICE_RULES = [
  { id: 'dining', name: '餐饮', icon: '🍲', keywords: ['餐饮', '吃饭', '美食', '餐厅', '外卖', '早餐', '午餐', '晚餐', '宵夜'] },
  { id: 'parking', name: '停车', icon: '🅿️', keywords: ['停车', '停车费', '停车场'] },
  { id: 'charging', name: '充电', icon: '⚡', keywords: ['充电', '充电费'] },
  // ... 更多分类
]
```

如果本地匹配不够准确，就调用 DeepSeek API：

```javascript
const parseTransactionFromText = async (text, categories) => {
  const prompt = `你是一个记账助手。请从以下文本中提取记账信息，返回纯 JSON 格式。

文本："${text}"

可用类别（必须返回 category ID，不是名称）：
${categoryList}

提取规则：
1. type: "expense"（支出）或 "income"（收入）
2. amount: 金额（纯数字，不要单位）
3. category: 从可用类别中选择最匹配的一个，必须返回 category 的 ID 值
4. date: 日期，格式为 YYYY-MM-DD
5. note: 备注信息，提取文本中的关键信息

今天日期：${new Date().toISOString().split('T')[0]}

只返回 JSON，不要任何其他文字或解释。`

  const response = await fetch('https://api.deepseek.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${apiKey}`
    },
    body: JSON.stringify({
      model: 'deepseek-chat',
      messages: [
        {
          role: 'system',
          content: '你是一个专业的记账助手，擅长从自然语言中提取结构化的记账信息。只返回 JSON 格式，不要其他内容。'
        },
        {
          role: 'user',
          content: prompt
        }
      ],
      temperature: 0.3,
      max_tokens: 500
    })
  })

  const data = await response.json()
  const content = data.choices[0]?.message?.content
  return JSON.parse(content)
}
```

DeepSeek 会从自然语言中提取出结构化的账单数据，包括 type、amount、category、date、note 这些字段。

### 3. 状态管理

用 Pinia 做状态管理，每个 store 负责特定的业务逻辑：

- **user.js**：用户登录状态和认证方法
- **transactions.js**：交易记录的 CRUD 操作和实时订阅
- **trips.js**：旅行记录的 CRUD 操作
- **categories.js**：分类管理，支持标准分类和旅行分类
- **settings.js**：系统设置管理，支持角色别名
- **voiceRules.js**：语音识别规则管理，支持关键词匹配

## 遇到的问题和解决方案

### 问题 1：语音识别准确率

Web Speech API 在安静环境下识别率还可以，但有时候会识别错。解决方案是结合本地规则和 AI 解析，本地规则能快速匹配常见场景（响应 < 100ms），AI 处理复杂语义（响应约 1-2s）。

### 问题 2：实时同步延迟

一开始数据更新有延迟，后来用了 Supabase 的实时订阅功能，数据变更后 < 500ms 就能推送到所有在线设备。

### 问题 3：移动端适配

刘海屏和底部横条会影响布局，用了 safe-area-inset-* 来适配，min-h-[100dvh] 来适配移动视口。

## 性能优化

### 代码分割

用 Vite 的代码分割功能，将第三方库分离到独立的 chunk：

```javascript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'echarts': ['echarts'],
        'supabase': ['@supabase/supabase-js'],
        'vendor': ['pinia', 'vue']
      }
    }
  }
}
```

### 其他优化

- 用 `v-show` 代替 `v-if` 减少组件销毁
- 用 `computed` 缓存计算结果
- 用 `debounce` 防抖处理用户输入
- 用 `scrollbar-hide` 类隐藏滚动条

## 总结

这个项目主要是为了解决家庭记账的同步问题，顺便尝试了语音记账和 AI 解析这些技术。用到的技术栈包括 Vue3、Vite、Pinia、Tailwind CSS、Supabase 和 DeepSeek API。

通过这个项目，对 Web Speech API 和大模型在实际场景中的应用有了更深的理解。语音记账虽然还有提升空间，但在日常使用中确实能减少记账的时间成本。
