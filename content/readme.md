# 📝 博客文章编写指南

欢迎来到你的内容创作空间！所有的文章都存放在 `content/posts/` 目录下，以 `.mdx` 格式结尾。

## 1. 文章结构

每篇文章由两部分组成：**Frontmatter（元数据）** 和 **Markdown 正文**。

### 第一部分：Frontmatter（元数据）

必须放在文件最顶部，用三个短横线 `---` 包裹。

```yaml
---
title: "这里是文章的标题"            # 必填：显示在页面最上方
date: "2025-12-30"                 # 必填：格式建议为 YYYY-MM-DD
category: "技术笔记"                # 选填：蓝色的小标签内容
readTime: "5 min"                  # 选填：预计阅读时间
description: "这是一段精炼的摘要..."  # 选填：显示在标题下方
tags: ["Nextjs", "React"]          # 选填：数组格式
---
```

### 2. 第二部分：Markdown 正文

在第二个 --- 之后开始书写你的正式内容。
```yaml
## 这是一个二级标题
### 这是一个三级标题
```

## 发布流程（重要）
目前你的文章列表页（/posts）使用的是手动配置模式。若要让新文章（如 test.mdx）出现在列表中，你需要完成以下步骤：

1. 创建文件：在 content/posts/ 下创建 test.mdx 并填写内容。

2. 更新列表配置：打开 app/posts/page.tsx 文件。

3. 修改数据源：在 ALL_POSTS 数组中手动添加该文章的元数据：
```
{
  id: 'test', // 必须与文件名一致
  title: '测试文章标题',
  date: '2025-12-30',
  excerpt: '这是文章的简短摘要...',
  category: '技术',
  readTime: '2 min'
}
```

提示：如果你希望以后“写完即发布”而不需要修改代码，建议将列表页的 ALL_POSTS 改为从 lib/posts.js 动态获取。

## 3. 进阶变量（自定义）

如果你想在页面上显示更多信息（例如文章作者、封面图链接），你可以直接在 Frontmatter 中添加：
```js
author: "土豆"
banner: "/images/post-cover.jpg"
```

注意：新增自定义变量后，你需要去 app/posts/[slug]/page.tsx 中修改代码，解构出这些变量并放置在你想要显示的 HTML 位置。

## 4. 存放路径建议

为了方便管理，建议统一命名规范：

- 文件路径：content/posts/my-new-post.mdx

- 访问地址：你的域名/posts/my-new-post

## 5. 写作小贴士 💡

1. 封面图：目前的页面设计上方有一个彩色渐变区域。如果你想显示真实图片，可以修改 page.tsx，将 <img> 标签的 src 指向你 Frontmatter 里的变量。

2. 暗色模式：写作时无需担心暗色模式，系统会自动处理文字和背景的对比度。

3. 表情符号：适量使用 Emoji 能够让技术文章看起来更有温度 🚀。
