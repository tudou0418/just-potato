# 太空飞翔土豆超人 - 设计方案

## 项目概述
将博客首页改造成基于Three.js的太空主题滚动视差页面，主角为披风土豆超人，创造沉浸式的太空飞行体验。

## 设计规范

### 颜色系统（基于现有主题）
- **浅色模式品牌色**：`--brand-base: #2563eb`（深空蓝）
- **暗色模式品牌色**：`--brand-base: #60a5fa`（明亮蓝）
- **背景色**：
  - 浅色：`--ui-surface: #fcfcfd`（极淡冷灰白）
  - 深色：`--ui-surface: #0f1115`（深蓝黑）
- **文本色**：
  - 主要：`--ui-text: #1a1c1e` / `#f1f5f9`
  - 次要：`--ui-text-muted: #64748b` / `#94a3b8`

### 排版系统
- 字重：800（标题）、700（重要）、600（链接）、500（正文）
- 字间距：-0.02em（标题）、-0.01em（正文）
- 动画：cubic-bezier(0.2, 0.8, 0.2, 1)

## 场景架构

### 1. 多层视差系统

| 层次 | 元素 | 移动速度 | 描述 |
|------|------|----------|------|
| 背景层 | 星空、星云 | 0.2x | 最远距离，营造深度感 |
| 中景层 | 星球、小行星带 | 0.5x | 中等距离，丰富场景 |
| 前景层 | 太空碎片、粒子 | 0.8x | 近距离，增加动感 |
| 主体层 | 土豆超人 | 1x + 独立动画 | 主角，跟随滚动 |

### 2. 土豆超人设计

**外观特征：**
- 身体：保持圆润土豆造型，增加肌肉感线条
- 头部：保留可爱特征，添加英雄式发型
- 披风：半透明飘逸材质，随飞行摆动
- 动作：飞行姿态，双臂张开微微前倾
- 表情：英勇自信的眼神，带微笑

**动画效果：**
- 呼吸：胸部轻微起伏
- 飞行：身体左右轻微摆动
- 披风：流动飘带效果
- 眨眼：周期性眨眼增加生动感

### 3. 太空场景元素

**背景层：**
- 深空渐变背景（深蓝→紫色→黑色）
- 动态星云效果，使用噪声函数
- 闪烁星星，不同大小和亮度

**中景层：**
- 地球、火星等星球轮廓
- 小行星带，随机分布
- 远方太空站剪影 *(TBD: 需要明确太空站样式和交互)*

**前景层：**
- 发光粒子效果
- 旋转的空间碎片
- 能量光环效果

## 技术实现方案

### 1. 组件架构

```typescript
// 主组件
<SpaceScene>
  <ScrollController />
  <SpaceBackground layer="background" speed={0.2} />
  <SpaceDecorations layer="midground" speed={0.5} />
  <SpaceDecorations layer="foreground" speed={0.8} />
  <SpacePotato />
  <ContentSections />
</SpaceScene>
```

### 2. 核心组件

#### SpaceScene
- 负责Three.js场景初始化
- 管理渲染循环
- 处理窗口大小调整

#### SpacePotato
- 土豆超人3D模型
- 独立动画系统
- 鼠标交互响应

#### ScrollController
- 监听滚动事件，计算滚动进度（0-1）
- 根据滚动进度计算各层的视差偏移量
- 触发场景动画和相机运动

#### SpaceBackground
- 星空背景生成
- 星云效果
- 星星闪烁

#### SpaceDecorations
- 动态生成装饰元素
- 管理粒子系统
- 处理动画效果

### 3. 性能优化

- 使用LOD系统，根据距离调整细节（距离>50m使用低模，距离<10m使用高模）
- 粒子数量动态控制（最多2000个粒子）
- 响应式画布，自动适配不同屏幕尺寸
- 内容分块加载，滚动时才加载对应区域的3D元素

### 4. 交互设计

#### 鼠标/触摸交互
- 桌面端：拖拽旋转视角，滚轮缩放
- 移动端：触摸滑动旋转，双指缩放
- 悬停：超人会有反应（如眨眼或手势）
- 滚动：根据滚动进度控制场景动画

#### 滚动同步
- 平滑的视差效果，各层移动速度按比例
- 页面内容淡入淡出效果
- 超人飞行轨迹与内容页面同步

## 实现步骤

1. **安装依赖**
   - three.js
   - @react-three/fiber (React Three Fiber)
   - @react-three/drei (辅助工具)
   - gsap (动画库，用于平滑过渡)

2. **创建基础场景**
   - 初始化Three.js场景
   - 添加相机和控制器
   - 创建基础光照

3. **实现土豆超人**
   - 使用Three.js基础几何体（BoxGeometry, SphereGeometry等）构建
   - 添加PBR材质和纹理
   - 实现基于GSAP的动画系统

4. **添加场景元素**
   - 创建星空背景
   - 添加星球和装饰
   - 实现粒子效果

5. **集成滚动交互**
   - 监听滚动事件
   - 计算视差效果
   - 同步内容动画

6. **优化和调试**
   - 性能测试和帧率监控
   - 移动端触摸交互优化
   - 用户体验细节调整

## 设计目标

- **视觉效果**：创造沉浸式太空体验
- **性能表现**：流畅的60fps动画，低端设备降级到30fps
- **用户体验**：自然流畅的交互，支持键盘和触摸
- **设计一致性**：符合现有博客风格，保持品牌识别度
- **移动端适配**：响应式设计，自动调整粒子数量和复杂度

## 重要注意事项

1. **性能优先**：3D效果可能会影响性能，需要提供降级方案
2. **渐进增强**：基础内容必须在没有3D的情况下也能正常访问
3. **可访问性**：确保键盘导航和屏幕阅读器支持
4. **SEO考虑**：3D内容可能影响SEO，需要权衡取舍
